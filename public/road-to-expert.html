What do these terms mean? Glossary

Every bit of jargon on this page, explained for a curious player — no statistics degree required. Dotted-underlined words above jump straight to their entry.

The bots
ISMCTS
Information Set Monte Carlo Tree Search — the candidate “Expert+” bot. It grows a tree of possible futures before each move while accounting for what it cannot see, like the opponent’s hidden hand. The number after the @ (as in ismcts@3000) is how many playouts it gets per decision.
flat Monte-Carlo (flat MC)
The bot Unbrewed ships today. For each legal move it plays a batch of quick random games to the end and picks the move with the best average outcome — no tree, no memory between decisions. “Flat” means every candidate move is tried the same way.
hard-64
The shipping “hard” difficulty of the flat Monte-Carlo bot: 64 playouts per candidate move. It is the yardstick — the gate arm asks whether the new tier beats exactly what players face today.
playouts (sims)
One playout is one fast simulated game played through to the end. More playouts means more thinking per decision. The counts on this page — like ismcts@3000’s 3,000 or hard-64’s 64 — are playouts per decision.
The experiments
arm
One experiment in the mission: a fixed pairing of two bots, run for a planned number of games. Each card above is one arm, and each isolates a different question — is it stronger, is it fair, is it worth the time.
the gate
The mission’s pass/fail bar. ISMCTS must win at least 60% of at least 1,000 games against hard-64, and the whole 95% interval has to sit above 50%. Clear the bar and the tier has earned the “Expert+” name.
knob grid / tuned flat MC
A sweep over the flat Monte-Carlo bot’s settings (its “knobs”) to find its strongest configuration. The winner becomes the “tuned flat MC” opponent — so ISMCTS is measured against the best non-tree bot we can build, not a weak default.
compute parity
A fair fight: both bots think the same total amount, so a win is credited to the smarter search, not to more thinking time. Without it, a critic could say the new bot only wins because it simulates more.
wall-clock matched
Instead of matching playout counts, match real seconds per move. It answers the practical question directly: given the same time on the clock, does the tree still make better moves?
mirror
A bot playing an identical copy of itself. Nobody is smarter, so the result must land near 50% — if it drifts away, the test setup itself is leaning on the scale, and every other arm becomes suspect.
pairing
Which deck faces which deck in a game — for example Mando vs Thrall, or King-Kong against itself. Games are spread evenly across an arm’s pairings so no single matchup dominates the average.
seat & seeding bias
Going first can matter, and so can the random seed a game starts from. The harness alternates which bot goes first and varies seeds so neither side gets a hidden head start. The mirror arm exists to prove this worked.
The statistics
win rate
Wins divided by games played, always shown from the ISMCTS side. 50% means dead even; the gate asks for 60%.
95% interval (Wilson)
The range where the true win rate plausibly lives, given how many games we have seen — wide when games are few, narrow as they pile up. We use the Wilson score interval, a textbook formula that stays honest even with small samples.
why the interval must exclude 50%
If the whole plausible range sits above 50%, luck alone can no longer explain the lead. That is the difference between “currently ahead” and “actually better” — and it is what the verdict lines above are checking.
warming up
Below 50 games a win rate jumps around too much to mean anything, so the charts show “warming up · n=…” instead of a headline number. It is not a verdict — just the page refusing to guess early.
in flight
Games the worker machines are playing right now — claimed but not yet finished. They appear in the pulse line on a running card and turn into completed games when they land.
decisions in
How far along an in-flight game is, counted in moves the bots have made so far. Long games can take a while, so this is the page’s heartbeat between finished results.
